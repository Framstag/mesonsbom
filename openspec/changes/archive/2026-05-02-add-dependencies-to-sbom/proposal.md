# Proposal: Add real dependency extraction to the SBOM generator

## Why

The current SBOM generated by `mesonsbom` only lists the main application component and omits the runtime libraries. This limits its usefulness for supply‑chain security and compliance scanning.

## What Changes

- Parse `intro-dependencies.json` produced by `meson introspect`.
- For each dependency, add a library component (name + version) to the CycloneDX SBOM.
- Record a dependency edge from the main application to each library.
- Update the integration test to verify that a library component appears in the SBOM.

## Capabilities

### New Capabilities
- `dependency-extraction`: Extract and include direct library dependencies from Meson build metadata.

### Modified Capabilities
- *None* – this change adds a new capability without altering existing spec behavior.

## Impact

- **Code changes**: `src/main.cpp` now reads `intro-dependencies.json` and adds components.
- **Tests**: `tests/integration_test.cpp` now checks for at least one `library` component.
- **No API breakage**: The command‑line interface remains unchanged.
